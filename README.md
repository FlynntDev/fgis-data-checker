# fgis-data-checker

**fgis-data-checker** — это инструмент для проверки и анализа данных, полученных из Федеральной государственной информационной системы (ФГИС). Проект предназначен для автоматизации процесса валидации, поиска ошибок и анализа данных, связанных с ФГИС.

## Возможности

- Загрузка и обработка данных из ФГИС
- Проверка корректности и целостности данных
- Выявление ошибок и несоответствий
- Генерация отчетов по результатам проверки

## Установка

1. Клонируйте репозиторий:
   ```sh
   git clone https://github.com/yourusername/fgis-data-checker.git
   ```
2. Перейдите в директорию проекта:
   ```sh
   cd fgis-data-checker
   ```
3. Установите необходимые зависимости (если используются):
   ```sh
   go mod tidy
   ```

## Использование

1. Подготовьте файл(ы) с данными ФГИС для проверки.
2. (Рекомендуется) Создайте или отредактируйте конфигурационный файл `config.yaml`, чтобы указать параметры проверки и пути к файлам.
3. Запустите программу:
   ```sh
   go run main.go [параметры]
   ```
   или соберите бинарник:
   ```sh
   go build -o fgis-data-checker
   ./fgis-data-checker [параметры]
   ```

## Работа с конфигом

fgis-data-checker использует YAML-файл конфигурации для задания параметров работы. Пример файла `config.yaml`:

```yaml
table1:
  name: "1.xlsx"
  sheet: "1"
  column: "B"
  header: "Номер в госреестре"

table2:
  name: "2.xlsx"
  sheet: "export_fif_arshin"
  column: "A"
  header: "numbersi"

output:
  name: "output.csv"
```

### Описание параметров

- `table1`, `table2` — описание таблиц для сравнения:
  - `name` — имя файла Excel
  - `sheet` — название листа
  - `column` — колонка для проверки
  - `header` — название столбца
- `output`:
  - `name` — имя файла для вывода результатов

По умолчанию программа ищет файл `config.yaml` в текущей директории.
